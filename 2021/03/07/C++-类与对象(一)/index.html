一.explicit,初始化列表

隐式转换与 explicit

关键词explicit可以使函数避免进行隐式转换
例如Time time=5; 对象time是不会等于5的，其中有隐式转换，加上explicit可以禁止掉。
一般对于单参数的构造函数，都声明为explicit，除非有特别原因。

二.inline,const,mutable,this,static

在类定义中实现函数inline：类内的成员实现其实也叫类内的成员函数定义

成员函数末尾的const

const : 常量，在成员函数后面加上一个const。不但要在函数声明中增加const，定义中也要。
作用 : 告诉系统，这个成员函数不会修改该对象里任何成员变量的值等等。
tip :假如 const Time time; 那么整个对象里的成员函数后面只要没有const，都不可以调用
即const对象只能调用const函数，非const对象两种都能调用

mutable

不稳定，容易改变的意思，const的反义词。mutable的引入也正好是为了突破const的限制
用mutable修饰成员变量。一个成员变量一旦被mutable修饰，就表示这个成员变量永远处于可以被修改状态，即便是在以const结尾的函数中，也可以被修改

2.虚函数与纯虚函数

纯虚函数：

调用虚函数执行的是”动态绑定”。动态:表示的就是在我们程序运行的时候才能知道调用了哪个子类的speak()虚函数。
动态的绑定到哪个子类，取决于new的是哪个
动态绑定:运行的时候才决定你的父类指针指向的是哪个子类函数

3.多态性

  • 多态性:只是针对虚函数说的
  • 体现在具有继承关系的父类和子类间，子类重定义(重写)父类的成员函数eat()，同时父类把这个eat()函数声明为virtual
  • 通过父类的指针，只有到了程序的运行期，找到动态绑定到父类指针上的对象，这个对象有可能是某个子类对象，也可能是父类对象，然后系统内部实际上是要查一个虚函数表，找到函数eat()的入口地址，从而调用父类或者子类的eat()函数，这就是运行时期的多态性

4.基类的析构函数一般写成虚函数(虚析构函数)

把父类的析构函数前面加上关键字virtual,当多态的子类对象销毁时，父类指针会调用子类对象的析构函数，然后执行父类的析构函数进行相关操作

七.基类与派生类关系的详细探讨

1.派生类对象模型简述

举个栗子:

1.一个是含义派生类自己定义的成员变量，成员函数的子对象
2.一个是该派生类所继承的基类的子对象，这个子对象中包含的是基类中定义的成员变量,成员函数(即派生类对象含有基类对应的组成部分)

  • 基类指针可以new派生类对象，因为派生类对象含有基类部分，所以我们是可以把派生类对象当成基类使用
  • 换句话说，我们可以用基类指针new一个派生类对象
  • 编译器帮助我们做了隐式的这种派生类到基类的转换
  • 这种转换的好处就是有些需要基类引用的地方你可以用这个派生类对象的引用来代替相应的，如果有些需要基类指针的地方，也可以用派生类对象的指针来代替

2.派生类构造函数

  • 既调用基类构造函数，又调用派生类构造函数。
  • 派生类实际上是使用基类的构造函数来初始化它的基类部分。你基类控制你基类部分的成员初始化，派生类控制派生类部分成员的初始化，大家各司其职。

4.不想当基类的类

c++ 11标准：final，加载类名后面，则有这个关键字的类就不能做基类

5.静态类型与动态类型

  • 静态类型:变量声明的时候的类型。静态类型编译的时候是已知的。

  • 动态类型:指的是这个指针/引用 所代表的(所表达的)内存中的对象的类型。这里是Son类型。

  • 动态类型，静态类型这种概念，只有基类指针/引用才存在这种静态类型和动态类型不一致的情况。

  • 6.派生类向基类的隐式类型转换

  • 编译器隐式的帮助我们执行了派生类到基类的转换

  • 这种转换之所以可以成功，是因为每个派生类对象都包含一个基类对象部分，所以基类的引用或指针是可以绑定到基类对象这部分

  • 不存在从基类到派生类的自动类型转换
    tips:个人理解，派生类对象比基类对象多了一点，所以派生类可以转基类，而基类到派生类是无中生有，所以不能转

    7.父类子类之间的拷贝与赋值

    基类的拷贝构造函数把派生类对象的基类部分拿去处理

  • 结论:用派生类对象为一个基类对象初始化或者赋值时，只有该派生类对象的基类部分会被拷贝或者赋值，派生类部分将被忽略掉

左值，右值，左值引用，右值引用，move

1.左值和右值

C++中所有的值都必然属于左值、右值二者之一。左值是指表达式结束后依然存在的持久化对象，右值是指表达式结束时就不再存在的临时对象。所有的具名变量或者对象都是左值，而右值不具名。
很难得到左值和右值的真正定义，但是有一个可以区分左值和右值的便捷方法：看能不能对表达式取地址，如果能，则为左值，否则为右值。

2.引用分类

三种形式的引用
1.左值引用(绑定到左值)

2.const引用(常量引用)

3.右值引用(绑定到右值):它是个引用

3.左值引用

左值引用:绑定到左值上

  • 引用没有空引用的说法，初始化时就必须绑定一个值
  • 左值不能绑定到右值，必须绑定到左值

4.右值引用

右值引用:就是引用右值，也就是说，绑定到右值
必须是绑定到右值的引用，
&&,希望用右值引用来绑定一些即将销毁的或者是一些临时的对象上

总结:

  • 返回值引用的函数，连同赋值，下标，解引用和前置递增递减运算符(–i)，都是返回左值表达式的例子
  • 返回非引用类型的函数，连同算术，关系，位以及后置递增运算符(i–)，都生成右值

    1.右值引用的引入目的

    (a)c++11引入，&&，代表一种新数据类型，引入新数据类型肯定有目的。
    (b)提高程序运行效率，把拷贝对象变成移动对象来提高程序运行效率。

    5.std::move函数

    c++11标准库里的新函数
    std::move :移动。实际上这个函数根本就没有做移动的操作
    std::move的能力只要一个：把一个左值强制转换为一个右值。带来的结果就是右值引用可以绑上去
